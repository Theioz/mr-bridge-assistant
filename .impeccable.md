## Design Context

### Users
Single user (the operator). Personal AI assistant, daily use across desktop and
mobile. Voice mode is a peer surface, not a fallback. The operator already runs
the system from a CLI and via voice — the web UI is the calm visual layer, not
the primary interaction.

### Brand Personality
**Quiet, precise, dependable.** The voice of a private chief-of-staff, not a
coach or a chatbot. Direct, structured, high-density, no filler, no emojis, no
motivational language. Quantify everything; present, don't celebrate.

### Aesthetic Direction
**Refined minimal.** Generous whitespace, restrained palette, a small set of
intentional typographic choices doing all the hierarchy work. Both light and
dark themes at full fidelity, system-following. Identical visual fidelity on
desktop and mobile — no amputated mobile experience.

**References:** Linear (restraint, considered motion, density without noise) and
Cultured Code's apps — Things 3, Bear (calm, generous whitespace, personal-app
warmth without sterility).

**Anti-references:**
- Generic SaaS dashboards — purple gradients, persistent sidebar nav,
  big-number metric cards, "AI-generated dashboard" baseline. Avoid.
- Crypto/trading terminal — neon greens and reds, monospace everywhere,
  glowing accents on pure black, "cockpit" excess. Avoid.

### Design Principles

1. **Information first, chrome last.** Cards only when grouping genuinely
   needs them. No decorative borders, no icon-above-heading templates, no
   metric-card cliché. If a section can stand on whitespace and typography
   alone, it should.

2. **Typography carries the hierarchy.** One distinctive sans for body and UI,
   one refined display or serif face for headings, dates, and key numerals.
   Tabular figures everywhere a number lives next to another number. At least
   a 1.25 ratio between type-scale steps; few sizes used with conviction.

3. **Whitespace is structural.** Negative space is the primary layout tool.
   Tight clusters within a section, generous separation between sections.
   Vary spacing for hierarchy — never uniform padding.

4. **Quiet motion only.** Page-load orchestration over scattered
   micro-interactions. Exponential easing (ease-out-quart/quint). No bounce,
   no elastic, no glow pulses. Reduced-motion fully respected.

5. **Same content, different rhythm.** Mobile is the same data laid out for
   one column with the same care, not a stripped desktop. Container queries
   for components; viewport queries only for the page-level rhythm.

6. **Restrained palette, tinted neutrals.** OKLCH throughout. Neutrals tinted
   subtly toward a single brand hue so light/dark surfaces feel cohesive. One
   accent color used sparingly (10% rule) — its rarity gives it power.

### Theme & Color Approach
- Both light and dark, system-following.
- No pure white, no pure black — always tinted.
- Brand hue picked once, every neutral tinted toward it (chroma 0.005–0.01).
- Single accent color reserved for: state changes, focus rings, the one thing
  on screen that requires attention. Not used for decoration.

### Typography Rules (project-specific)
- Avoid Inter, Roboto, system defaults, and all of Impeccable's reflex-reject
  list (Fraunces, Playfair, Crimson, Cormorant, Outfit, Plus Jakarta, DM Sans,
  Instrument, IBM Plex, Space Grotesk, etc.).
- Pair: one distinctive sans (body + UI) + one refined display/serif (headings,
  numerals, timestamps).
- Tabular figures (`font-variant-numeric: tabular-nums`) on every metric.

### Motion Rules (project-specific)
- One orchestrated entrance per page load (staggered section reveals,
  ~30–60ms stagger, total under 500ms).
- Hover/focus transitions: 120–180ms, ease-out-quart.
- Never animate layout properties; transform/opacity only.
- `prefers-reduced-motion: reduce` strips all non-essential motion.

### Accessibility
- WCAG AA contrast minimum on both themes. AAA where it doesn't fight the aesthetic.
- 44px minimum touch targets on mobile.
- Visible focus ring on every interactive surface (use the accent color).
- Reduced motion respected (see motion rules).
